## 작업 요약

Phase 1 셋업 이후, 팀 엠블럼을 사이트에 적용하고 화면 확인용 샘플 데이터 시드
명령을 추가했다. 개발 서버에서 데이터가 정상 렌더링되는 것까지 확인.

## 1. 로고(엠블럼) 적용

- `fcskylogo.png`(검은 배경 흰 마크, "sky F.C 2007" + 별)를 사이트 브랜딩 자산으로 판단.
  사용자 업로드용 `media/`가 아니라 정적 파일 위치 `static/img/fcskylogo.png`로 이동.
- `base.html`:
  - 네비게이션 바 브랜드에 로고 추가(검은 배경이라 흰 배경 패딩 처리해 파란 네비에서 가독성 확보).
  - `<link rel="icon">`로 파비콘 지정.
- `home.html`: 히어로 영역 중앙에 로고(140px) 배치.
- 검증: `/static/img/fcskylogo.png` → 200 image/png, 홈에서 로고 참조 정상.

## 2. 샘플 데이터 시드 명령

`apps/teams/management/commands/seed.py` 관리 명령 추가.

- 실행: `python manage.py seed` / 초기화 후 재생성: `python manage.py seed --flush`
- `get_or_create` 기반 → 여러 번 실행해도 중복 없음(idempotent).
- 경기 일시는 `오늘 기준 상대값(D-28 ~ D+11)`으로 생성 → 홈의 "다가오는 경기 / 최근 결과"가 항상 채워지도록.
- 종료 경기에는 우리 팀 득점 수만큼 `MatchEvent`(GOAL) 생성, 공격수/미드필더에 배분.

### 생성 규모

| 항목 | 수 |
|------|----|
| 시즌 | 1 (2026, 현재) |
| 팀 | 3 (K7 / 40 / 50) |
| 선수 | 19 |
| 대회 | 4 (구청장기·협회장기·K7리그·시민리그) |
| 상대팀 | 7 |
| 경기 | 11 (종료 8 + 예정 3) |
| 경기 이벤트 | 10 |
| 입상 | 3 |

## 3. 검증

- `seed` 실행 결과 위 규모대로 생성됨.
- 홈: 다가오는 경기/최근 결과 박스 채워짐.
- 팀 목록: 3팀 노출. K7 상세: 선수 8명 표 렌더링.
