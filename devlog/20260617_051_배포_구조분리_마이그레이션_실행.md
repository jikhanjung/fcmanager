# 20260617_051 — 배포 구조 분리 1회 마이그레이션 실행 (dolfinid)

devlog 050에서 설계·커밋한 "개발 소스 ↔ 운영 런타임 `/srv/FcSky` 분리"의
**1회 마이그레이션을 dolfinid에서 실제로 실행**했다.

## 실행 전 상태 (확인)

- `/srv/FcSky/.env` **없음**, `/srv/FcSky/docker-compose.yml`·`deploy.sh` **없음**
  → `sync_to_srv.sh` 미실행, `.env` 백업이 계속 skip 되던 원인.
- 운영 컨테이너 `deploy-web-1`(image `0.5.7`)이 **개발 체크아웃**
  `/home/honestjung/projects/FcSky/deploy` 에서 기동 중(옛 구조).
- `/srv/FcSky/scripts/backup_db.py`(hourly)만 정상 존재.

## 실행 절차

```bash
cd ~/projects/FcSky && ./deploy/sync_to_srv.sh          # /srv 로 운영 파일 복사
cp -p deploy/.env /srv/FcSky/.env
echo "IMAGE_TAG=0.5.7" >> /srv/FcSky/.env               # 런타임 위치로 .env 이전 + 태그
cd ~/projects/FcSky/deploy && docker compose -p deploy down   # 옛 컨테이너 정리 (다운 시작)
/srv/FcSky/deploy.sh 0.5.7                              # pull → 스냅샷 → up → 헬스체크
curl -fsS -o /dev/null -w "%{http_code}\n" http://127.0.0.1:8003/FcSky/admin/login/
```

## 결과 (검증)

- ✅ `/srv/FcSky/.env` 생성(99B = `DJANGO_SECRET_KEY` + `IMAGE_TAG=0.5.7`) → 백업 대상으로 잡힘.
- ✅ `/srv/FcSky/`에 `docker-compose.yml`·`deploy.sh`·`scripts/backup_db.py` 모두 동기화됨.
- ✅ 컨테이너 전환: `deploy-web-1`(옛 구조) 제거 → **`fcsky`**(`/srv` 런타임, image `0.5.7`) 기동.
- ✅ `deploy.sh`가 pre-deploy DB 스냅샷 자동 생성(롤백 안전망):
  `/srv/FcSky/backup/pre_deploy/fcsky_pre_deploy_0.5.7_20260617_015544.sqlite3` (376K).
- ✅ 사이트 정상: `/FcSky/admin/login/` `200`, 공개 홈 `/FcSky/` `200`.
- 다운타임 약 7초(헬스체크 기준, 그동안 nginx 502 → maintenance.html).

이후 운영 기동은 `/srv/FcSky` 에서만, `~/projects/FcSky` 는 순수 git 소스로 유지.
표준 배포는 `sync_to_srv.sh` → `/srv/FcSky/deploy.sh X.Y.Z` (devlog 050 / 매뉴얼 `deploy.md`).
