# SQLite WAL 전환 (0.6.19)

- 날짜: 2026-07-14
- 선행: devlog 088(디렉터리 마운트 — WAL 형제 파일 호스트 공유 기반)·089(gosu — 소유권).
  088 에서 "fcmanager 는 delete 저널, WAL 전환은 후속 과제"로 남긴 것을 마무리.

## 변경

- `config/settings.py` DATABASES OPTIONS (cdGTS 동형):
  - `init_command: 'PRAGMA journal_mode=WAL;'` — 모든 커넥션에서 보증(WAL 자체는 DB 파일에
    영속이지만 명시가 안전).
  - `timeout: 20` — writer 경합 흡수(기본 5s → 20s).
  - `transaction_mode: 'IMMEDIATE'` — 쓰기 트랜잭션이 시작 시점에 잠금 확보, 업그레이드
    데드락(SQLITE_BUSY) 회피.
- 효과: gunicorn 워커 3개 + 중계 콘솔의 동시 읽기/쓰기에서 reader 가 writer 에 막히지 않음.

## 주변 인프라는 이미 WAL-ready (088 에서 준비됨)

- 디렉터리 마운트 → `-wal`/`-shm` 이 호스트 `/srv/fcmanager/db/` 에 존재.
- hourly `backup_db.py` = sqlite3 online backup API — WAL 포함 일관 스냅샷.
- pre-deploy 스냅샷·rollback restore·daily 미러 = wal/shm 동반 복사 기구현.
- 롤백 호환: WAL 은 파일에 영속이나 구버전 이미지도 같은 sqlite3 라이브러리라 무해.
