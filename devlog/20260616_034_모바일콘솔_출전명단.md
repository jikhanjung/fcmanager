# 20260616_034 — 모바일 중계 콘솔 개편 + 경기별 출전 명단

> 이어지는 작업: [20260616_033_실시간중계_콘솔_폴링.md](20260616_033_실시간중계_콘솔_폴링.md)

폰에서 몇 탭으로 중계 입력하는 모바일 UI로 콘솔을 개편하고, 그 기반이 되는
**경기별 출전 명단(라인업)** 기능을 추가했다.

## 1. 모바일 중계 콘솔 (`/matches/<pk>/live/` 개편)

상호작용: **동작 먼저 → 선수**(바텀시트). 데스크톱 폼 → 엄지 친화 UI로 교체.

- 상단 고정바: 🔴 LIVE / 예정 / 종료 상태 + **분 자동 입력**(킥오프 경과, 탭하면 수정) + 시작·종료 토글
- 큰 두 버튼: **⚽ 우리 득점 / ⚽ 상대 득점**
  - 우리 득점: 바텀시트에서 득점 선수 탭 → 도움 선수(또는 `도움 없음`) → 저장 (3탭)
  - 상대 득점: **1탭** 즉시 +1
- 보조: 🟨 경고 · 🟥 퇴장(선수 1탭, 시트에 `상대팀 선수` 버튼) · 🔄 교체(나간→들어온, SUB_OUT+SUB_IN)
- **등번호 타일** 그리드(번호 없으면 이름만, 주장 Ⓒ, 벤치 표시)
- **최근 기록**: 도움을 득점과 한 줄로 표시, `✕` 한 번으로 즉시 취소
- 모든 동작 **새로고침 없이 fetch**로 즉시 반영(스코어·기록 갱신)

### 백엔드
- `_live_payload()`로 폴링(`live.json`)과 콘솔 AJAX 응답이 동일 payload 공유.
- 콘솔 POST는 `X-Requested-With: fetch`면 JSON 반환, 아니면 302 리다이렉트(폴백).
- `serialize_timeline()` 항목에 `id` 추가(삭제 버튼용).
- 교체용 `action="sub"`(SUB_OUT + SUB_IN) 추가.

## 2. 경기별 출전 명단 (라인업)

`Player.squad`(임시 필드 주석에 "추후 로스터 모델로 분리 예정")가 가리키던 자리.
범위는 **선발/벤치 + 주장**으로 합의(포지션은 선수 기존값 사용, 포메이션 제외).

- **모델 `MatchLineup`**(match, player, role=STARTER/BENCH, jersey_number, is_captain),
  `unique_together(match, player)`. admin에 인라인 등록.
- **모바일 편집 화면 `/matches/<pk>/lineup/`**(staff): 선수마다 `선발/벤치/제외`
  세그먼트(`.btn-check` 라디오, JS 프레임워크 없음) + 주장 Ⓒ 라디오 1명. 상단에 선발·벤치 카운트.
  저장 시 기존 명단 교체, 등번호는 소속(TeamMembership)에서 채움.
- **콘솔 연동**: `_build_roster()`가 라인업이 있으면 그것을(선발 먼저·벤치 뒤, 주장 표시)
  쓰고, 없으면 팀 전체 소속으로 폴백 → 비파괴적.
- 상세·콘솔에서 "🧑‍🤝‍🧑 출전 명단" 링크.

## 검증 (테스트 클라이언트, 롤백)

- 콘솔: GET 렌더(roster/timeline json_script·offcanvas), AJAX 시작/득점+도움/교체/삭제
  즉시 JSON 응답, 비-AJAX 302 폴백.
- 라인업: GET 폼, 저장(선발4·벤치2·주장1) → 콘솔 로스터 6명(벤치·주장 표시) →
  라인업 삭제 시 팀 전체(35명) 폴백.
- `manage.py check` 무이슈, 마이그레이션 `matches.0005_matchlineup`.

## 배포 / 후속

- 이미지 `0.3.0 → 0.4.0`.
- 후속 후보: 공개 경기 페이지에 선발 라인업 표시, 교체 시 IN=벤치·OUT=출전 중으로 제한,
  로스터(시즌/대회 단위 등번호) 관리 화면, `Player.squad` 임시 필드 정리.
