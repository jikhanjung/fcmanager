# 20260617_059 — SaaS Phase C: 클럽별 권한 + 온보딩

> 선행: Phase B(라우팅+스코핑). 상위: [P03](20260617_P03_SaaS_멀티테넌트_전환계획.md).

## 한 일

### 클럽별 운영진 권한 (전역 is_staff → ClubMembership)
- **`apps/clubs/permissions.py`**: `is_club_staff(user, club)`(superuser=전 클럽) +
  `club_staff_required` 데코레이터(비로그인→로그인, 비운영진→403).
- 각 앱 뷰의 `staff_required` 를 `club_staff_required` 로 교체(모든 `@staff_required` 사용처가
  현재 클럽 기준으로 전환).
- **context processor `apps.clubs.context_processors.club`**: `current_club`·`is_club_staff` 제공.
- 템플릿의 `user.is_staff` → `is_club_staff` 로 교체(8개 템플릿). Django Admin 링크만
  `user.is_staff`(Django 스태프) 유지.
- **base.html navbar**를 `current_club` 유무로 가드 — 플랫폼(테넌트 밖) 페이지에서 테넌트
  `{% url %}` 가 깨지지 않도록.

### 온보딩 (플랫폼)
- **플랫폼 랜딩**(`/`): 클럽 목록 + 내 클럽 + 로그인/클럽 만들기. (기존 단일 클럽 리다이렉트 대체)
- **플랫폼 로그인/로그아웃**(`/accounts/login|logout/`, 로그인 후 랜딩): 테넌트 없이도 로그인 가능.
- **클럽 생성**(`/clubs/new/`, 로그인 필요): `ClubForm`(이름·slug 자동·예약어/중복 검사) →
  Club 생성 + 생성자 `ClubMembership(OWNER)`. 미들웨어 `clubs` 세그먼트 예약.
