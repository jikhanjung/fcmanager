# 대회 상세에서 참가팀·경기 웹 관리

2026-07-12

## 목표

참가팀(CompetitionEntry)과 경기 생성/삭제는 그동안 admin 전용이었다. 대회 상세
화면(운영진)에서 직접 참가팀 추가/편집/삭제, 경기 추가/삭제, 경기 정보(일시·장소·부문)
편집까지 가능하게 한다.

## 변경 내용

### 참가팀 관리 (apps/competitions)

- `CompetitionEntryForm` (forms.py): 우리 팀(select) 또는 외부팀 이름(text) 중
  정확히 하나 입력. 외부팀 이름은 기존 `Opponent` 와 일치하면 연결, 없으면 자동
  생성(`get_or_create`) — 외부팀을 admin 에서 미리 만들 필요가 없어졌다.
  Opponent 는 클럽 간 공유되므로 이름을 rename 하지 않고 항상 이름으로 찾거나 만든다.
  같은 대회·부문 중복 참가는 폼에서 거부.
- 뷰: `entry_add` / `entry_edit` / `entry_delete` (`@staff_required`).
  - URL: `manage/competitions/<slug>/entries/...`
  - 삭제는 POST 전용. entry 에 연결된 경기가 있으면 삭제 차단(경기 CASCADE 방지).
  - `?division=<id>` 로 부문 프리셀렉트(패널 탭에서 진입 시).
- 템플릿: `competitions/entry_form.html` (외부팀 이름 datalist 자동완성 포함).

### 경기 추가/삭제 (apps/matches + competitions)

- `MatchCreateForm` (matches/forms.py): 부문·단계·홈/원정 참가팀·일시·장소·상태·비고.
  참가팀/부문 선택지는 해당 대회로 제한, 드롭다운 라벨에 부문 표기(`이름 (40대부문)`).
  홈=원정 거부. 스코어·이벤트는 저장 후 '결과 편집'에서.
- 뷰 `match_add` (competitions/views.py): `manage/competitions/<slug>/matches/add/`.
  club·competition 은 서버에서 주입.
- 뷰 `match_delete` (matches/views.py): 확인 페이지(`match_confirm_delete.html`,
  이벤트·영상 수 경고) 후 삭제, 대회 상세로 복귀. `matches/<pk>/delete/`.

### 경기 정보 편집 확장

- `MatchResultForm` 에 `kickoff`(datetime-local)·`venue`·`division` 필드 추가.
  부문 선택지는 경기의 대회로 제한. `match_edit.html` 에 필드 배치 + '경기 삭제' 버튼.
- 참가팀 드롭다운 라벨 공통 헬퍼 `_entry_label` / `_competition_entries` 로 정리.

### 대회 상세 패널 (`_competition_panel.html`)

- staff 에게 패널(부문 탭)마다 **+ 참가팀 추가 / + 경기 추가** 버튼(부문 프리셀렉트).
- 출전팀 배지에 ✎ 편집 / ✕ 삭제(confirm 후 POST) 인라인 컨트롤.
- 조별리그 경기 행·녹아웃 카드(준결승/결승)에 **편집** 버튼 — 우리 경기는
  `matches:edit`, 상대팀 간 경기는 `matches:opponent_edit`(?next= 복귀).
  비운영진에게는 기존 공개 UI 그대로.

## 검증

- `manage.py check` 통과, 기존 테스트 50건 통과.
- 스모크 테스트(테스트 DB, Client): 참가팀 추가(우리 팀/외부팀 자동 생성)·중복/XOR
  검증·편집·경기 연결 시 삭제 차단, 경기 추가(동일 팀 거부)·정보 편집(일시/장소/부문)·
  삭제, 비운영진 403, 대회 상세 staff 버튼 렌더까지 전부 확인.

## 남은 것 (admin 전용 유지)

입상(Award)·출전명단은 웹 편집 화면 별도(출전명단은 콘솔용 `match_lineup` 존재),
클럽 멤버십·부문 시간 오버라이드는 admin 유지.
