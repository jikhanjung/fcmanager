# 20260620_066 중계 콘솔 전후반 구분 + 전후반 길이 설정

## 배경
중계 입력 화면에 전·후반 구분이 없어 시계가 단조 증가만 했고, 이벤트가 어느 하프에
속하는지 알 수 없었다. 전후반 길이는 대회/부문마다 다르므로(예: 50대부문 25분) 설정 가능해야 한다.

## 결정 사항 (사용자 확정)
- **길이 설정 위치**: 대회(Competition) 기본값 + 부문(Division) 오버라이드.
  경기는 부문값 우선 → 없으면 대회값으로 해석.
- **후반 시계**: 이어서 카운트. 전반 30분이면 후반은 30:00부터 흐르고 이벤트 분도 연속 표기(예: 52분).
- **전환 UX**: 단계 버튼 흐름 — `전반 시작 → 하프타임 → 후반 시작 → 종료`. 하프타임엔 시계 정지.

## 변경
### 모델
- `Competition.half_length_minutes`(기본 45).
- `Division.half_length_minutes`(null=대회값 사용).
- `Match.period`(SCHEDULED/FIRST/HALFTIME/SECOND/FINISHED) + `second_half_started_at`.
  기존 `live_started_at`은 전반 시작 시각으로 의미 정리.
- `Match.half_length_minutes`/`half_length_seconds` 프로퍼티(부문→대회 해석).
- `MatchEvent.half`(1/2) — 콘솔 기록 시 진행 단계에서 채움.
- 마이그레이션: competitions 0009, matches 0015.

### 뷰 (`apps/matches/views.py`)
- `_handle_live_action`: 단계 액션 `start_first`/`halftime`/`start_second`/`finish`로 전환.
  (구 `start`은 `start_first`로 하위호환.) 이벤트 생성 시 `half=_current_half(match)` 태깅.
- `_elapsed_seconds`: 진행 단계별 경과 초(전반=now-시작, 하프타임=길이 고정,
  후반=길이+후반경과, 종료=풀타임/길이).
- `_live_payload`: `period`/`period_display`/`half_length` 추가. 공개 폴링·콘솔 공유.
- 콘솔 컨텍스트에 `half_length` 전달. select_related에 competition/division 추가.

### 템플릿
- `match_live.html`: 진행 단계 배지 + 단계 진행 버튼(STAGE 맵), 시계는 FIRST/SECOND에서만
  흐름, 전후반 길이 표시. `data-period`/`data-half-length` 추가.
- `match_detail.html`: 시청자 폴링 힌트에 현재 단계(전반/하프타임/후반) 표시.

### Admin
- Competition/Division 목록·인라인에 `half_length_minutes` 노출.
- Match에 '중계 진행'(period·시작시각) 섹션, MatchEvent에 `half` 노출.

### 프론트 대회 관리 편집(`/manage/competitions/<slug>/edit/`)
- `CompetitionForm`에 대회 `half_length_minutes` 필드 추가(연도 아래).
- 부문별 길이 오버라이드는 폼에서 노출하지 않음(드물어 불필요 판단). 모델 필드는 유지하고
  필요 시 Admin 부문 인라인에서 편집. `sync_divisions`는 부문 생성/삭제만 하고
  기존 부문의 길이 오버라이드는 보존.

## 추가: 시계 일시정지/재개 (matches 0016)
- `Match.paused_at`(정지 시각) + `Match.paused_seconds`(현재 하프 누적 정지 초).
- 콘솔에 하프타임·종료와 별개의 **⏸ 일시정지 / ▶ 재개** 버튼(전·후반 진행 중에만 노출).
  부상·지연 등으로 하프 전환 없이 시계만 잠깐 멈췄다 이어갈 때 사용.
- `_elapsed_seconds`: 정지 중이면 `paused_at` 기준으로 계산하고 하프 누적 정지(`paused_seconds`)를 뺀다.
  재개 시 멈췄던 시간을 `paused_seconds`에 더하고 `paused_at`을 비운다.
- 하프/경기 시작·종료 전환은 `paused_at`을 비우고, 하프 시작은 `paused_seconds`도 0으로 초기화.
- `_live_payload`에 `paused` 추가. 시청자 페이지 힌트도 '전반 (일시정지) …' 표시.
- Admin '중계 진행' 섹션에 `paused_at`·`paused_seconds` 노출.
