# 20260615_003 — Phase 2 공개 조회 페이지

## 목표
Phase 1에서 구축한 모델·Admin 위에 방문자용 공개 조회 페이지를 구현한다.
일정/결과, 경기 상세, 리그 순위표, 득점 순위, 명예의 전당, 선수 상세.

## 환경 정리 (선행 작업)
- `~/venv/FcSky` 에 의존성 미설치 상태였음 → `pip install -r requirements.txt`
  (Django 5.1.15, Pillow 12.2.0, asgiref, sqlparse).
- 신규 DB에 `migrate` 적용, `seed` 로 샘플 데이터 생성
  (팀 3·선수 19·대회 4·경기 11·이벤트 10·입상 3).

## 구현 내용

### 뷰 / URL
- **matches**: `urls.py` 신규.
  - `matches/` `schedule` — 경기 목록 + 팀·대회·시즌·상태(예정/종료) 필터.
  - `matches/<pk>/` `match_detail` — 스코어 + MatchEvent 타임라인.
  - `matches/scorers/` `scorers` — GOAL/OUR 이벤트 선수별 집계.
- **competitions**: `urls.py` 신규.
  - `standings/` `standings` — 선택 리그의 우리 팀 전적 자동 집계
    (경기·승·무·패·득점·실점·득실·승점, 승점=승×3+무×1). FINISHED·점수 입력분만.
  - `awards/` `awards` — Award를 시즌별로 `regroup`.
- **teams**: `players/<pk>/` `player_detail` 추가 — 프로필 + 소속 이력 +
  득점/도움/경고/퇴장 집계(`Count`+`Q` 조건 집계) + 득점 기록 + 수상.
- `config/urls.py` 에 matches·competitions URLconf include.

### 모델
- `Match.get_absolute_url()`, `Player.get_absolute_url()` 추가.

### 템플릿
- `matches/match_list.html`, `match_detail.html`, `scorers.html`
- `competitions/standings.html`, `awards.html`
- `teams/player_detail.html`
- `base.html` 네비게이션에 일정·결과 / 순위표 / 득점 순위 / 명예의 전당 추가.
- 홈·팀 상세에서 경기/선수 상세로 링크 연결.

## 설계 메모 / 한계
- **순위표**: 데이터 모델상 상대팀 간 경기는 기록하지 않으므로(상대는 `Opponent`),
  완전한 리그 순위가 아니라 **FC Sky 팀들의 리그 전적**만 산출. 화면에 명시.
- 필터는 GET 파라미터 + `onchange` 자동 제출(별도 JS 빌드 없음, CLAUDE.md 관례 준수).
