# DB 파일 마운트 → 디렉터리 마운트 전환 (0.6.16)

- 날짜: 2026-07-14
- 동기: cdGTS·fsis2026 과 동형화. 파일 마운트(`/srv/fcmanager/db.sqlite3:/app/db.sqlite3`)는
  컨테이너가 만드는 WAL 형제 파일(`-wal`/`-shm`)이 **컨테이너 사설 레이어에 갇혀** 호스트에
  안 남는다 → 호스트 백업(hourly `backup_db.py`·m710q daily 미러)이 최신 커밋을 못 보는
  스테일 읽기 위험. 디렉터리 마운트면 형제 파일이 호스트에 함께 산다.

## 패턴 선택

- cdGTS: `/srv/cdGTS` **루트 전체** → `/app/hostdb`.
- fsis2026: `/srv/fsis2026/db` **DB 전용 하위 디렉터리** → `/app/hostdb`.
- **fsis2026 패턴 채택** — `.env`·backup·배포 스크립트를 컨테이너에 노출하지 않는다.

## 변경

- compose(운영 `deploy/host/`·로컬 `deploy/`): `- /srv/fcmanager/db:/app/hostdb` +
  `environment` 에 `DATABASE_PATH: /app/hostdb/db.sqlite3` **고정**(env_file 보다 우선이라
  `.env` 오배선 무해 — 구 DEPLOY.md 의 ".env DATABASE_PATH 주의" 계열 함정 자체가 소멸).
- `deploy.sh`: down 직후 **1회 자동 레이아웃 이행**(`$ROOT/db.sqlite3` → `$ROOT/db/`, wal/shm
  포함 — writer 없는 시점이라 mv 안전). 스냅샷 소스·DB 게이트 기대값(`/app/hostdb/db.sqlite3`) 갱신.
- `rollback.sh`: 복원 대상 `$ROOT/db/db.sqlite3` (스냅샷 출처 레이아웃과 무관하게 정본 위치로).
- `scripts/backup_db.py`(hourly cron): 새 경로 + **legacy fallback**(이행 전 cron 실행 대비).
- m710q `~/scripts/backup-fcmanager.sh`(daily 미러, repo 밖): scp `db/db.sqlite3` + 구 레이아웃
  fallback. dev_data 미러 경로는 변화 없음(파일명 동일).
- `deploy.toml` db_path=`/srv/fcmanager/db`, README/DEPLOY.md 현행화.
