# 076 — 경기 결과 편집에서 홈/원정팀 선택 추가

## 배경

운영진 경기 편집 화면(`/matches/<pk>/edit/`)에서 홈팀·원정팀(`home_entry`/`away_entry`)을
바꿀 수 없었다. 참가팀 지정은 admin 에서만 가능했다. 결과 입력 화면에서 팀까지
바로잡을 수 있게 해 달라는 요청.

## 변경

- **폼** `MatchResultForm`:
  - `home_entry`·`away_entry` 필드 추가(맨 앞). 둘 다 `required=True`.
  - 선택지(queryset)는 **이 경기의 대회(`competition`)에 등록된 참가팀**으로 제한
    (`__init__` 에서 instance.competition_id 로 필터). 다른 대회 참가팀은 노출 안 함.
  - 드롭다운 라벨은 기본 `__str__`('이름 - 대회') 대신 참가팀 이름만(`label_from_instance`).
  - `clean()`: 홈팀과 원정팀이 같으면 검증 에러.
- **편집 템플릿** `match_edit.html`: 스코어 카드 상단에 홈팀/원정팀 셀렉트 추가.
  팀이 가변이 됐으므로 득점·승부차기 라벨을 고정 팀명 대신 "홈팀/원정팀"으로 변경.

## 확인

- `python manage.py check` 무결, `apps.matches` 테스트 23개 통과.
- 임시 데이터로 폼 검증: 대회 외 참가팀 제외(3개만 노출)·라벨(이름만)·필수·동일팀 거부·교체 통과 확인.

## 참고

- 점수는 홈/원정 기준으로 유지되므로, 홈/원정을 바꾸면 기존 점수도 함께 검토해야 한다(운영진 책임).
- 대진 자동 진행(feeder)으로 away_entry 가 자동 설정되는 경기는 수동 변경이 덮어쓸 수 있음에 유의.
