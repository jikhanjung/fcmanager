# 20260616_032 — 경기 편집 입력칸 폭 조정 (분·설명)

v0.2.10~v0.2.13에 걸쳐 경기 결과 편집 이벤트 표의 입력 폭을 다듬은 작업 정리.

## 문제
- 분(minute) 컬럼이 `width:5rem`(약한 힌트)이라, `min-width`를 가진 옆 컬럼들에 밀려
  그 아래로 짓눌려 숫자가 가려짐.
- 반대로 화면이 넓을 땐 분·설명(입력칸)이 남는 폭을 다 흡수해 과하게 늘어남
  (셀렉트 컬럼은 내용 크기로 고정이라 안 늘어남).

## 조치 (templates/matches/match_edit.html, apps/matches/forms.py)
- 분 컬럼 헤더: `width:5rem` → **`min-width:5rem`** (못 줄어드는 하한 → 짓눌림 방지).
- 분 입력: **`max-width:6rem`**, 설명 입력: **`max-width:24rem`** (과확장 방지, 남는 폭은 셀렉트로).
- 좁은 화면에선 `width:100%`가 살아 자연 축소.

## 교훈
- 표 컬럼 폭을 지키려면 `width`(권장값, 줄어들 수 있음)보다 **`min-width`**(하한)가 안전.
- 입력칸 과확장은 입력 요소에 **`max-width`** 로 상한.

## 배포
- 최종 `honestjung/fcsky:0.2.13`.
