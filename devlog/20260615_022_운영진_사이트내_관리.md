# 20260615_022 — 운영진 사이트 내 관리 (팀·선수·공지 CRUD)

Django admin 의존을 줄이고 운영진(staff)이 **사이트 안에서** 직접 관리하도록 CRUD 추가.
모두 `@user_passes_test(is_staff)` 게이트(익명/비운영진은 로그인으로).

## 팀 (apps/teams)
- `TeamForm` — 이름·slug(비우면 ASCII 자동, 폴백 team-N)·연령대·창단일·로고·소개.
- 뷰: `team_create`(`teams/add/`), `team_edit`(`teams/<slug>/edit/`).
- **삭제는 제외**(연쇄 삭제 위험) — 추가/수정만. 진입: 팀 목록 "+ 팀 추가", 팀 상세 "팀 수정".

## 선수 (apps/teams)
- `PlayerForm`(이름·출생연도·포지션·구분·사진·프로필) + `MembershipForm`(등번호·주장).
- 뷰: `player_add`(`teams/<slug>/players/add/`) → Player + TeamMembership(현재 시즌),
  `player_edit`(…/edit/), `player_remove`(…/remove/).
- **선수 "삭제" = 팀에서 제외**(TeamMembership만 삭제, 선수 레코드·득점/도움 기록 보존).
- 진입: 팀 상세 "+ 선수 추가", 각 선수 카드의 "수정/제외"(staff).

## 공지 (apps/notices)
- `NoticeForm`(제목·내용·상단고정·게시). 뷰: `notice_create/edit/delete`.
- 목록·상세에서 운영진은 미게시 공지도 열람(배지 표시). 진입: "+ 공지 작성", 상세 "수정/삭제".

## 공통
- 폼 템플릿은 체크박스를 `form-check`로 분기 렌더. 저장 시 `messages` 알림.

## 테스트
- test Client로 팀 생성 / 선수 추가·수정·제외(기록 보존) / 공지 작성·수정·삭제 전부 302,
  익명 접근 302(로그인). 운영진 버튼·폼 노출, 익명 미노출 HTTP 확인.

## 배포
- 운영진 관리 기능 완성 기념 **v0.2.0** (`honestjung/fcsky:0.2.0`).
