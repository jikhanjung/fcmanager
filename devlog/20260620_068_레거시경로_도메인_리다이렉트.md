# 20260620_068 — 레거시 경로/도메인 → 새 도메인 301 리다이렉트 (dolfinid nginx)

> 멀티테넌트 SaaS 전환 후 옛 진입점(서브패스·구 도메인)을 새 전용 도메인으로 정리.
> **대상 파일은 호스트 nginx 설정**(`/etc/nginx/sites-available/2026biennale`, repo 비추적) —
> 이 devlog 가 기록. 설정은 hourly 백업의 nginx tar 에 자동 포함됨.

## 배경

`fcmanager.app`(SaaS) 와 `biennale.app` 가 각각 전용 도메인으로 가동 중인데, 옛 진입점이
여전히 살아 있었다:

- `34.64.158.160/FcSky/` → 레거시 fcsky 컨테이너(8003) 직접 프록시
- `34.64.158.160/2026biennale/` → `/srv/2026biennale` 정적 직접 서빙
- `biennale.nopeoplestime.info` → `http://34.64.158.160/2026biennale/` (구 도메인, IP 경유)

## 변경 (default_server `server_name _` + 구 도메인 블록)

세 경우 모두 **새 도메인으로 301 영구 리다이렉트**(경로·쿼리 보존):

| 진입점 | → 리다이렉트 |
|---|---|
| `/FcSky`, `/FcSky/<path>?<q>` | `https://fcmanager.app/fcsky/<path>?<q>` |
| `/2026biennale[/<path>]`, `/biennale2026[...]` | `https://biennale.app/<path>` |
| `biennale.nopeoplestime.info/<path>` | `https://biennale.app/<path>` (이전: IP/2026biennale 2단 경유) |

핵심 디렉티브:

```nginx
# /FcSky → fcmanager.app/fcsky (경로 보존)
location = /FcSky { return 301 https://fcmanager.app/fcsky/; }
location /FcSky/  { rewrite ^/FcSky/(.*)$ https://fcmanager.app/fcsky/$1 permanent; }

# /2026biennale·/biennale2026 → biennale.app (경로 보존)
location = /biennale2026 { return 301 https://biennale.app/; }
location = /2026biennale { return 301 https://biennale.app/; }
location ~ ^/(?:2026biennale|biennale2026)/(.*)$ { return 301 https://biennale.app/$1; }

# 구 도메인 직접 → biennale.app
server { server_name biennale.nopeoplestime.info; return 301 https://biennale.app$request_uri; }
```

## 검증

```
/FcSky/matches/            → 301 https://fcmanager.app/fcsky/matches/
/FcSky/standings/?team=us  → 301 https://fcmanager.app/fcsky/standings/?team=us
/2026biennale/a/b.html     → 301 https://biennale.app/a/b.html
biennale.nopeoplestime.info/foo/bar → 301 https://biennale.app/foo/bar
```

`nginx -t` 통과 후 reload. 다른 사이트(biennale.app HTTPS, fcmanager.app) 무영향.

## 메모

- 레거시 fcsky 컨테이너(8003)·`/srv/FcSky` 는 **아직 가동·보존**(롤백 안전망). 공개 진입만 리다이렉트로 차단.
- 백업: 수정 전 원본 `/etc/nginx/sites-available/2026biennale.bak-fcsky-redirect-20260620`.
- `biennale.nopeoplestime.info` 의 `/2026biennale/...` 경로는 biennale.app 의 자체 블록이 한 번 더
  정리(→ root) — 흔한 루트 진입엔 영향 없음.
