# 20260617_055 — 대회 참가팀(Entry) 전면 개편 R1~R3

> 계획: [P05](20260617_P05_대회참가팀_Entry_리팩터.md). Match 를 참가팀(entry) 쌍으로, OpponentMatch 흡수, Opponent 공유화.

## 한 일 (R1 모델·마이그레이션 / R2 서비스 / R3 뷰·템플릿)

### 모델
- **`CompetitionEntry` = 참가팀**: `team`(우리) XOR `opponent`(외부), `(competition,division,*)` unique.
  `name`/`club_id` 프로퍼티.
- **`Match` = entry 쌍**: `our_team`/`opponent` → `home_entry`/`away_entry`,
  `our_score`/`opponent_score` → `home_score`/`away_score`, `is_home` 제거, `kickoff` nullable,
  `opponent_feeder` → self(Match). **OpponentMatch 흡수**(상대팀 간 경기도 일반 Match).
- **호환 프로퍼티**(churn 방어): `our_entry`·`our_team`·`opponent`·`our_score`·`opponent_score`·
  `is_home`·`result`·`is_our_match`·`winner_entry` 를 home/away+club 에서 도출 → 뷰·템플릿 대부분 무변경.
- **`Opponent` 공유화**: `club` FK 제거, `name` 전역 unique.

### 마이그레이션 (matches 0013, 한 파일 내 add→backfill→remove)
- 기존 Match: `is_home` 으로 home/away entry·score 배정. OpponentMatch → Match(상대팀 간) 변환.
- 검증: 7 Match + OM 1 = **8 Match**, entry 11(team 4·opponent 7), 호환 프로퍼티 정확.

### 서비스·시그널
- `finished_matches`/`club_record` home/away 기반, `recompute_score` our_entry 기준 home/away 저장.
- `signals` 피더: `winner_entry`로 결승 away_entry 자동 기입.

### 뷰·템플릿
- 표준순위: **부문(division)별** 참가팀 집계로 재작성(우리·상대팀 간 경기 통합).
- 대회 상세 대진표/패널: home/away entry symmetric 표시.
- 일정 목록은 우리 팀 entry 참가 경기만. `admin:` reverse → 플랫폼 `/admin/` 하드코딩(테넌트 밖).

## 검증
- `check` 무이슈. 공개 13 + 운영진 4 페이지 **전부 200**.
- 쓰기: 중계 콘솔 득점 → 점수 재집계(our 1:1, home 1:1) 정상.
- 테스트: `apps.matches`·`apps.competitions` **통과**.

## 남은 작업 (R4/R5)
- `seed`·`load_seocho_knockout` 명령을 새 모델로 갱신(현재 구 필드라 실행 시 오류).
- `apps.teams` 스모크 테스트(seed 의존 + 테넌트 URL) 갱신.
- 폼/템플릿 라벨 정리(`우리/상대` → home/away 명시), `opponent_match_edit.html`(om.home/away) 갱신.
- 이후 SaaS B2(쿼리 스코핑).
