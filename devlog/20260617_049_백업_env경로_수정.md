# 20260617_049 — 백업 `.env` 경로 수정 (m710q 적용 안내)

## 배경

m710q daily pull 백업(`backup-fcsky.sh`)이 매번 `WARN: .env 없음 (건너뜀)` 로
`.env` 를 건너뛰고 있었다. devlog 036 메모엔 "운영 `/srv/FcSky` 에 `.env` 가
없다" 고 적혀 있었으나, 실제로는 **위치를 잘못 보고 있었던 것**이다.

## 원인

- 운영의 `.env` 는 `docker-compose.yml` 의 `env_file: - .env`(compose 파일 기준
  상대경로)로 주입된다 → 실제 파일은 **`/srv/FcSky/deploy/.env`**.
- 백업 스크립트는 `${REMOTE_PATH}/.env`(= `/srv/FcSky/.env`)를 scp 하고 있어
  항상 실패 → best-effort skip.

## 수정 (repo)

`scripts/backup-fcsky.sh` 의 `.env` pull 경로를 `deploy/` 하위로 정정.

```diff
-scp -q "${REMOTE}:${REMOTE_PATH}/.env" "${CURRENT_DIR}/.env" 2>/dev/null && \
+scp -q "${REMOTE}:${REMOTE_PATH}/deploy/.env" "${CURRENT_DIR}/.env" 2>/dev/null && \
     log ".env 복사 완료" || log "WARN: .env 없음 (건너뜀)"
```

## m710q 에서 할 일 (배포 + 검증)

> 운영(dolfinid)은 손댈 것 없음. m710q 의 배포본 스크립트만 갱신하면 된다.

```bash
# 1) repo 최신 가져오기 (이미 받는 경로가 있으면 그걸로)
cd <FcSky repo on m710q> && git pull

# 2) 배포본 스크립트 갱신
cp scripts/backup-fcsky.sh ~/scripts/backup-fcsky.sh
chmod +x ~/scripts/backup-fcsky.sh

# 3) 수동 검증 실행
~/scripts/backup-fcsky.sh --full-snapshot

# 4) 결과 확인 — ".env 복사 완료" 가 찍혀야 정상
grep "\.env" ~/backups/FcSky/backup.log | tail -1
ls -la ~/backups/FcSky/current/.env
```

`~/backups/FcSky/current/.env` 가 생기고 로그에 `.env 복사 완료` 가 보이면 끝.
이후 cron(`0 4 * * *`)은 그대로 두면 매일 자동 포함된다.
