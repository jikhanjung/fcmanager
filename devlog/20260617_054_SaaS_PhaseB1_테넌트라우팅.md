# 20260617_054 — SaaS Phase B1: 테넌트 라우팅

> 계획: [P04](20260617_P04_SaaS_PhaseB_라우팅_스코핑.md) · 범위: 라우팅만(쿼리 스코핑은 B2~).

## 한 일

경로 `/<club-slug>/` 로 테넌트를 해석하고, 기존 뷰·템플릿을 **한 줄도 안 바꾸고** 클럽
하위로 라우팅되게 했다(`set_script_prefix`).

- **`apps/clubs/middleware.py` `TenantMiddleware`**: 경로 첫 세그먼트로 `Club` 조회 →
  `request.club` 설정 + `path_info` 에서 슬러그 제거 + `set_script_prefix("/<slug>/")` +
  `request.urlconf = "config.urls_tenant"`. 플랫폼 세그먼트(admin/accounts/static/media)·
  루트는 통과, 알 수 없는 슬러그는 404.
- **URLconf 분리**: `config/urls.py`(플랫폼: admin·랜딩) ↔ `config/urls_tenant.py`(클럽 앱·로그인).
- **`platform_home`**: 루트(`/`)는 활성 클럽으로 리다이렉트(현재 fcsky 1개).
- settings: `TenantMiddleware` 를 SessionMiddleware 뒤·CommonMiddleware 앞에 등록.

## 검증 (test client)

- `/` → 302 `/fcsky/` · `/fcsky/`·`/fcsky/matches/`·`/fcsky/matches/<id>/`·`/fcsky/stats/` → 200
- `/admin/login/` → 200(플랫폼) · `/nope/` → 404(알 수 없는 클럽)
- 렌더된 링크에 `/fcsky/` 자동 부착(`{% url %}` 무변경) 확인.
