# FC Sky 웹사이트 — 프로젝트 계획

- 작성일: 2026-06-14
- 문서 종류: 계획 (Plan)

## 1. 배경 / 목표

**FC Sky**의 공식 웹사이트를 구축한다. 현재 팀 구성:

- **K7 팀** (20-30대)
- **40대 팀**
- **50대 팀**

출전 대회/리그:

- 구청장기 대회
- 협회장기 대회
- K7 리그 (K7 팀)
- 시민리그 (K7 팀)

웹사이트가 제공해야 할 기능:

- 팀별 **선수 구성** 입력·조회
- **참가 리그/대회** 관리
- **경기 스케줄 및 결과** 관리
- **입상 내역** 관리
- (장기) **실시간 중계**

## 2. 기술 스택 결정

| 항목 | 선택 | 비고 |
|------|------|------|
| 백엔드 | Django 5.1, Python 3.12 | 데이터 모델이 명확하고 Admin 활용도가 높음 |
| 프론트엔드 | Django Template + Bootstrap 5 | 동호회 사이트에 가장 빠르고 유지보수 쉬움 |
| DB | SQLite(개발) → PostgreSQL(운영) | |
| 실시간(예정) | Django Channels + Redis | Phase 4 |
| 배포(예정) | Gunicorn + Nginx 또는 PaaS | Phase 5 |
| 가상환경 | `~/venv/FcSky` | |

## 3. 도메인 모델 설계

핵심 개념: **누가(선수) / 어느 팀에서 / 어떤 대회에 / 언제 경기하고 / 결과는 / 무엇을 받았나**.

| 모델 | 앱 | 설명 |
|------|----|----|
| Team | teams | 팀 (K7 / 40대 / 50대) |
| Player | teams | 선수 |
| TeamMembership | teams | 선수↔팀 소속 (시즌·등번호·주장) |
| Season | competitions | 시즌 (연도 집계) |
| Competition | competitions | 대회/리그 (리그·토너먼트·컵) |
| CompetitionEntry | competitions | 팀의 대회 출전 (시즌별) |
| Award | competitions | 입상/수상 내역 (팀 또는 선수) |
| Opponent | matches | 상대팀 (외부 팀) |
| Match | matches | 경기 (일정 + 결과, 승/무/패 자동 판정) |
| MatchEvent | matches | 경기 이벤트 (골·도움·카드·교체) → 실시간 중계 기반 |

> 설계 포인트: `MatchEvent`를 1단계부터 모델로 심어 두어 Phase 4 실시간 중계가 자연스럽게 확장되도록 한다. 등번호·주장은 선수가 아니라 `TeamMembership`(시즌 단위)에 두어 시즌·팀 이동을 표현한다.

## 4. 단계별 로드맵

### Phase 1 — 기반 + 데이터 입력 (MVP 코어)
- Django 프로젝트/앱 셋업, 모델 정의, 마이그레이션
- Django Admin 커스터마이징으로 전 데이터 입력 가능하게
- 한글 로케일, 미디어(사진) 업로드, 기본 페이지

### Phase 2 — 공개 조회 페이지 (방문자용)
- 홈, 팀 소개, 선수단(Squad), 일정 & 결과, 순위/기록, 명예의 전당
- 대회별·팀별 필터

### Phase 3 — 운영 편의 기능
- 회원/권한(운영진 로그인), 공지사항·갤러리
- 통계 자동 집계(득점왕, 출전 수, 승/무/패), 시즌 아카이브

### Phase 4 — 실시간 중계
- MatchEvent 실시간 입력 → 화면 자동 갱신 (Channels/WebSocket)
- 라이브 스코어보드, 타임라인

### Phase 5 — 배포 & 운영
- PostgreSQL 전환, 정적파일, 배포(Gunicorn+Nginx/PaaS), 도메인·HTTPS·백업
