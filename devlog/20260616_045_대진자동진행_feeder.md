# 20260616_045 — 대진 자동 진행(feeder): 준결승 결과 → 결승 자동 갱신

## 배경

"준결승 결과 입력하면 결승 상대 자동" 요청. 단, **결승 상대는 우리 준결승이 아니라 반대편
준결승 승자**(우리는 우리 경기만 보유)라, 반대편 준결승까지 기록·연결해야 자동화 가능.
→ "대진 연결(feeder)로 완전 자동" 선택.

## 한 일

### 모델
- `OpponentMatch.stage`(녹아웃 표기) + `winner` 프로퍼티(승자 Opponent, 무승부/미입력 None).
- `Match.opponent_feeder`(→OpponentMatch): 이 경기 상대 = 그 경기 승자(예: 반대편 준결승).
- `Match.advance_feeder`(→Match self): 그 경기(우리 준결승)에서 이겨야 이 경기 진출.
- 마이그레이션 0011.

### 자동 전파 (signals.py, post_save)
- OpponentMatch 저장 → `feeds_opponent_of` 결승들의 상대를 승자로 자동 기입.
- Match 저장 → `advances_to` 경기들: 우리가 지면 결승 `CANCELLED`(+note '준결승 탈락…'),
  이기면 `SCHEDULED` 복구.
- `apps.py ready()`에서 시그널 등록.

### 데이터 연결 (load_seocho_knockout)
- 결승마다 반대편 준결승(OpponentMatch, stage SF: 매현/신반포, 서초/ACI) 생성 +
  `opponent_feeder`·`advance_feeder` 연결. 멱등.

### admin
- OpponentMatch: stage·search_fields. Match: feeder 2개 autocomplete + '대진 자동 진행' 섹션.

## 검증 (DB 사본)

- load_seocho: 결승 2건 대진 연결, 반대편 준결승 생성.
- 반대편 준결승(매현 3:1 신반포) 입력 → **결승 상대 자동 '매현'**.
- 우리 준결승 패배(0:2) → 결승 **CANCELLED + '준결승 탈락' note**. 승리 정정(2:0) → **SCHEDULED 복구**.
- `check` 무이슈.

## 배포

- 0.5.3 배포(마이그레이션 0011 자동 적용). **운영에 load_seocho 재실행**해 양 부문 결승 feeder 연결:
  40대 결승←(매현/신반포)·(스카이 40대 vs 양재), K7 결승←(서초/ACI)·(스카이 K7 vs 서초크루).

## 운영 사용법

- 반대편 준결승 결과는 admin '상대팀 간 경기'에서 입력 → 결승 상대 자동.
- 우리 준결승은 평소처럼 경기 편집/라이브 콘솔로 결과 입력 → 지면 결승 자동 취소.
- 우리 팀 경기만 보유하므로 대진표는 '우리 경로' 중심.
