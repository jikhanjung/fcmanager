# 20260617_050 — 배포 구조 분리 (개발 소스 ↔ 운영 런타임)

## 문제

운영(dolfinid)에서 컨테이너를 **개발 소스 체크아웃에서 직접** 기동하고 있었다.
`/home/honestjung/projects/FcSky/deploy/` 의 `docker-compose.yml`·`.env` 로
`docker compose up`(프로젝트명 "deploy" → `deploy-web-1`). 즉 git 체크아웃 = 런타임.

→ 개발 소스와 운영 런타임이 섞여 위험. 기동에 필요한 파일은 `/srv/FcSky/` 에 두고,
배포 때만 소스에서 거기로 복사하는 구조가 바람직.

## 설계 (fsis2026 패턴 차용, FcSky 규모로)

세 책임 분리:

| 단계 | 스크립트 | 위치/실행 |
|---|---|---|
| 빌드·푸시 | (수동/`build.sh`) | m710q: 버전 bump → `docker build` → `docker push` |
| 운영 파일 동기화 | `deploy/sync_to_srv.sh` | dolfinid: `git pull` 후 실행 → `/srv/FcSky/` 로 복사 |
| 컨테이너 교체 | `/srv/FcSky/deploy.sh X.Y.Z` | dolfinid: pull → IMAGE_TAG 갱신 → down → **pre-deploy 스냅샷** → up → 헬스체크 |

### 런타임 레이아웃 (`/srv/FcSky/`)

```
/srv/FcSky/
├── db.sqlite3, media/, backup/, maintenance/   (기존)
├── scripts/backup_db.py     ← sync (hourly cron)
├── docker-compose.yml       ← sync (deploy/host/, build 없이 IMAGE_TAG pull)
├── deploy.sh                ← sync (deploy/host/)
└── .env                     ← 호스트 관리(비밀+IMAGE_TAG). sync 대상 아님
```

- 운영 compose(`deploy/host/docker-compose.yml`)는 `build:` 없이
  `image: honestjung/fcsky:${IMAGE_TAG}` 로 pull. `container_name: fcsky`, `restart: unless-stopped`.
- 로컬/개발 빌드는 기존 `deploy/docker-compose.yml`(build 포함) 그대로 사용.
- `.env` 는 `/srv/FcSky/.env` 로 이전(런타임 위치). 백업 스크립트 `REMOTE_ENV_PATH` 도 이 경로로 정정.
- 배포 중 down~up 짧은 공백은 nginx 502 → `maintenance.html` 자동 노출(devlog 023).

## 표준 배포 절차

```bash
# m710q: 빌드·푸시
docker build -f deploy/Dockerfile --build-arg DJANGO_URL_PREFIX=FcSky \
  -t honestjung/fcsky:X.Y.Z . && docker push honestjung/fcsky:X.Y.Z

# dolfinid: 동기화 + 교체
cd ~/projects/FcSky && git pull
./deploy/sync_to_srv.sh
/srv/FcSky/deploy.sh X.Y.Z
```

## 1회 마이그레이션 (현 운영 → 새 구조)

```bash
# dolfinid 에서
cd ~/projects/FcSky && git pull && ./deploy/sync_to_srv.sh
# .env 를 런타임 위치로 이전 + IMAGE_TAG 추가
cp -p deploy/.env /srv/FcSky/.env && echo "IMAGE_TAG=0.5.7" >> /srv/FcSky/.env
# 기존(개발 소스 기동) 컨테이너 정리 후 새 위치에서 기동
cd ~/projects/FcSky/deploy && docker compose -p deploy down
/srv/FcSky/deploy.sh 0.5.7
# 검증
curl -fsS -o /dev/null -w "%{http_code}\n" http://127.0.0.1:8003/FcSky/admin/login/
```

이후 `/home/honestjung/projects/FcSky` 는 순수 git 소스로만 두고, 운영 기동은 `/srv/FcSky` 에서만.

## 변경 파일 (repo)
