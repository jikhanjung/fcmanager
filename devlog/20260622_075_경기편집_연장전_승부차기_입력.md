# 075 — 경기 결과 편집 화면에 연장전·승부차기 입력 추가

## 배경

운영진 경기 편집 화면(`/matches/<pk>/edit/`)에서 연장전·승부차기 결과를 입력할
방법이 없었다. 승부차기 점수 필드(`home_pso_score`/`away_pso_score`)는 모델·중계
콘솔·admin 에만 있었고, 수동 편집 폼(`MatchResultForm`)엔 노출되지 않았다.

추가로 참가팀(Entry) 개편(d49f026)에서 폼 필드가 `our_score`/`opponent_score` →
`home_score`/`away_score` 로 리네임됐으나 `match_edit.html` 템플릿은 옛 이름을
계속 참조하고 있어 **점수 입력칸이 렌더링되지 않는 잠재 버그**가 있었다.

## 변경

- **모델**: `Match.went_to_extra_time`(연장전 진행) BooleanField 추가.
  연장 골은 이미 최종 스코어에 합산되므로 별도 점수 필드 없이 '연장 접전' 표시 플래그로만 둔다.
  마이그레이션 `0018_match_went_to_extra_time`.
- **폼** `MatchResultForm`: `went_to_extra_time`·`home_pso_score`·`away_pso_score` 필드 추가.
  `clean()` 검증 — 승부차기는 (1) 양 팀 모두 입력하거나 모두 비울 것, (2) 본점수 동점일 때만 인정.
- **편집 템플릿** `match_edit.html`:
  - 깨진 점수 입력칸 수정(`form.our_score`→`form.home_score`, 홈/원정 기준 라벨).
  - "연장전 · 승부차기" 카드 섹션 신설(연장전 체크박스 + 홈/원정 승부차기 점수 + 안내문).
- **상세 템플릿** `match_detail.html`: 연장전 진행 시 "연장전"/"연장" 배지 표시
  (승부차기로 갈린 경우는 기존 승부차기 표시 유지).
- **admin**: 결과 fieldset 에 `went_to_extra_time` 추가.

## 확인

- `python manage.py check` 무결, `apps.matches` 테스트 23개 통과.
- 폼 검증: 한쪽만 입력/비동점 승부차기 → 에러, 동점+양쪽 입력 → 통과 확인.
