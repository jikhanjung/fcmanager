# 20260616_038 — 팀 선수 추가를 '기존 멤버 선택'으로 변경

## 배경

팀 페이지(`/teams/<slug>/`)의 **+ 선수 추가**가 새 `Player`를 만드는 폼으로 연결돼,
같은 사람이 팀마다 중복 Player로 생성되는 원인이었다([[20260616_037]]에서 중복 3건 병합).
`Player`는 멤버 마스터로 따로 관리하고, 팀 추가는 **기존 Player에서 선택**해 소속만
만들도록 변경.

## 한 일

- `apps/teams/forms.py` — `MembershipAddForm` 신설.
  - 필드: `player`(선택), `jersey_number`, `is_captain`. **새 Player 생성 없음.**
  - `player` 선택지는 전체 Player에서 **이미 그 팀·시즌 명단에 있는 선수 제외**(중복 소속 방지).
  - `clean()`에서 중복 소속 재검증.
- `apps/teams/views.py` `player_add` — `PlayerForm`+`MembershipForm`(신규 생성)
  → `MembershipAddForm`(기존 선택)으로 교체. 선택한 Player로 현재 시즌 소속 생성.
- `templates/teams/player_add.html` 신규 — 선수 select + 등번호/주장 + "명단에 없으면
  새 선수 등록(관리자: `admin:teams_player_add`)" 안내. 기존 `player_form.html`은 선수 *수정*
  (`player_edit`)에서 계속 사용.

## 검증

- 운영 DB 사본 + staff 테스트 클라이언트:
  - add 페이지 200, 선수 `<select>`·"새 선수 등록" 링크 렌더.
  - sky-50 이미 소속 선수는 선택지에서 제외(59명 중 16명 제외 → 43 옵션).
  - 기존 Player(강민우) 선택 POST → 302, 소속 16→17건 생성, **Player 총수 59 불변(신규 생성 안 함)**.
- 0.4.4 배포 후 라이브에서 add 페이지 200, navbar v0.4.4.

## 메모

- 새 인물 등록은 관리자(`/admin/teams/player/`)에서 Player 생성 → 팀 페이지에서 선택.
  추후 온사이트 'Player 신규 생성' 흐름이 필요하면 별도 작업.
