## 테스트 target (cdGTS `[targets.test]` 동형)

- 빌드 호스트 m710q 가 곧 테스트 서버. **운영과 동일 레이아웃 `/srv/fcmanager`**(스크립트
  `ROOT`·compose 볼륨 하드코딩과 일치) — docker 컨테이너 경유로 생성(uid 1000 소유).
- `deploy/host/deploy-dev.sh` 신설(DEPLOY_SNAPSHOT=0), self-heal 추출·`sync_to_srv.sh` 에 포함.
  매니페스트 `[targets.test]`(host=m710q, 0.0.0.0:8005, snapshot=false) 추가.
- `.env`: HOST_PORT=**8005**(8003 은 m710q 에서 fsis 컨테이너 점유), dev 전용 SECRET_KEY.
- DB·media = `~/dev_data/fcmanager/`(운영 백업 미러, daily 갱신) **복사본** — 폐기 가능.
  runserver 기반 `scripts/run-testserver.sh`(데이터 확인용)와 별개로, 이쪽은 **배포
  파이프라인 자체**를 시험하는 용도. dev_data 는 daily 로 갱신되지만 `/srv/fcmanager` 복사본은
  자동 갱신 안 됨 — 필요 시 `cp ~/dev_data/fcmanager/db.sqlite3 /srv/fcmanager/` 후 재기동.
- compose 의 `DJANGO_ALLOWED_HOSTS` 는 environment 로 고정(fcmanager.app·localhost·127.0.0.1 등)
  — LAN IP 로 폰 접근하려면 compose 수정 필요(현 테스트 용도엔 불필요, smoke 는 127.0.0.1).
