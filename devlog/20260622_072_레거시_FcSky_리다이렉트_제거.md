# 20260622_072 — 레거시 `/FcSky/` nginx 리다이렉트 제거

repo 리네임(FcSky→fcmanager) 마무리의 일환. devlog 068 에서 레거시
`/FcSky/` → `fcmanager.app/fcsky/` 301 리다이렉트를 호스트 nginx 에 두었으나,
리네임 정리 차원에서 운영(dolfinid) 에서 해당 블록을 제거.

## 변경 (운영 호스트 dolfinid, repo 비추적)

- 파일: `/etc/nginx/sites-available/2026biennale`
- 제거한 블록:
  ```nginx
  # ── FC Sky (Django 컨테이너, 127.0.0.1:8003) ──
  location = /FcSky { return 301 https://fcmanager.app/fcsky/; }
  location /FcSky/ { rewrite ^/FcSky/(.*)$ https://fcmanager.app/fcsky/$1 permanent; }
  ```
- 백업: `/etc/nginx/sites-available/2026biennale.bak-fcsky-removed-20260622`
- `sudo nginx -t` 통과 → `systemctl reload nginx`.

## 검증

- `GET /FcSky/` (Host=IP, Host=_) → **404** (이전 301). default_server 의
  `location / { return 404; }` 가 처리.
- `GET /foobar/` → 404 (회귀 없음). biennale 리다이렉트(`/2026biennale/` 등)는 그대로 동작.
- `https://fcmanager.app/fcsky/` → 200 (신규 도메인 정상).

## 레거시 fcsky 컨테이너(8003) 폐기

리다이렉트 제거로 컨테이너가 어떤 nginx 블록에서도 참조되지 않게 되어 같은 날 폐기.

- `docker stop fcsky && docker rm fcsky` (restart=unless-stopped → stop 시 자동재시작 안 됨).
- 마운트는 전부 bind(`/srv/FcSky/db.sqlite3`, `/srv/FcSky/media`), 익명/네임드 볼륨 없음
  → `rm`(without `-v`)로 호스트 데이터 보존.
- 검증: 8003 포트 해제, `/srv/FcSky` 데이터·`.env`·DB·media 전부 잔존, `fcmanager.app/fcsky/` 200.
- **보존**: `/srv/FcSky` 데이터 디렉터리 + 이미지 `honestjung/fcsky:0.5.7` (콜드백업 — 필요 시 재기동 가능).

## 레거시 fcsky 백업 정리

컨테이너 폐기로 레거시 `/srv/FcSky` 를 별도로 백업할 이유가 사라짐.

- dolfinid hourly 는 **별도 cron 2개**가 돌고 있었음:
  - 신규 `/srv/fcmanager/scripts/backup_db.py` — 트랙 `fcmanager`(DB) + `dolfinid_nginx`(설정 tar)
  - 레거시 `/srv/FcSky/scripts/backup_db.py` — 트랙 `fcsky`(DB) + `dolfinid_nginx`
  컨테이너 폐기로 후자가 불필요 → **2026-06-22 crontab 에서 제거**(직전 crontab 백업
  `/tmp/crontab.bak-20260622`). `dolfinid_nginx` 백업은 신규 스크립트가 동일하게 커버 →
  손실 없음. 레거시 스크립트(`/srv/FcSky/scripts/backup_db.py`)·기존 스냅샷(`/srv/FcSky/backup/`)은
  콜드백업으로 보존.
- m710q daily 의 **구 cron `backup-fcsky.sh`(04시)** 는 신규 `backup-fcmanager.sh`(05시)와
  병행 가동 중이었음(둘 다 동일 스크립트, 경로만 `/srv/FcSky` vs `/srv/fcmanager`).
  레거시 폐기로 04시 백업이 불필요해져 **2026-06-22 정리**:
  - crontab 에서 `0 4 * * * .../backup-fcsky.sh` 줄 제거(직전 crontab 백업:
    `~/scripts/crontab.bak-before-fcsky-removal-20260622`).
  - 스크립트는 `~/scripts/_retired/backup-fcsky.sh.retired-20260622` 로 이동(삭제 대신 보관).
  - `~/backups/FcSky/` 기존 스냅샷은 콜드백업으로 보존.
- `/srv/FcSky` 데이터·`honestjung/fcsky:0.5.7` 이미지는 콜드백업으로 보존하므로,
  롤백이 필요하면 그 시점 스냅샷으로 충분(정기 백업 대상에서 제외).
- 문서: `docs/operation_manual/backup.md` 의 트랙명 표기를 `fcsky` → `fcmanager` 로 정정
  (코드 `SOURCES` 는 이미 `fcmanager`).

## 영향

- 옛 `/FcSky/...` 북마크·외부 링크는 이제 404 (의도된 폐기).
- 운영 호스트에서 더 이상 fcsky 관련 가동 리소스 없음(컨테이너·포트·nginx 참조·백업 모두 정리).
