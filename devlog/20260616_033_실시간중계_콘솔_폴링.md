# 20260616_033 — 실시간 중계 콘솔 + 폴링 자동 갱신 (Phase 4 MVP)

> 계획: [20260616_P02_실시간중계_MVP.md](20260616_P02_실시간중계_MVP.md)

## 한 일

WebSocket/Redis 없이 **폴링 기반**으로 경기 실시간 중계 MVP를 구현했다.

### 운영진 중계 콘솔 — `/matches/<pk>/live/` (staff 전용)
- 모바일 친화 화면. **LIVE 시작 / 경기 종료** 토글.
- 빠른 입력 폼(일반 POST, JS 빌드 없음):
  - ⚽ 우리 득점(선수 + 도움 선택) / 상대 득점 +1
  - 🟨 경고 · 🟥 퇴장 · ⏱ 교체 IN/OUT · 자책골 (종류·팀·선수 선택)
  - 분(minute)은 킥오프 경과시간으로 자동 채움(수정 가능)
- 입력된 이벤트 목록 + 개별 삭제(확인 후).
- 득점/자책골/삭제 시 **점수 자동 재집계**(`recompute_score`).

### 공개 폴링 엔드포인트 — `/matches/<pk>/live.json`
- `{status, is_live, our_score, opponent_score, result, timeline[], updated_at}`.
- 타임라인은 기존 득점-도움 페어링 로직(`build_timeline`)을 재사용해 직렬화.

### 공개 경기 상세 — LIVE 자동 갱신
- LIVE면 🔴 LIVE 배지 + "실시간 자동 갱신 중…" 안내.
- 바닐라 JS가 `live.json`을 **7초 주기**로 폴링 → 스코어·타임라인 갱신.
- 경기 종료가 감지되면 폴링 중지 후 1.5초 뒤 새로고침(최종 승/무/패 배지 반영).
- 운영진에게 "🔴 중계 콘솔" 버튼 노출.

### 기타
- 일정·결과 목록에서 LIVE 경기에 🔴 LIVE 배지.
- `base.html`에 `{% block extra_js %}` 추가.

## 설계 메모

- **점수 단일 출처는 `Match.our_score`/`opponent_score`**. 중계 콘솔에서 이벤트를
  추가/삭제할 때 집계로 재계산·저장 → 순위표·결과·득점 순위 등 기존 화면과 일관.
  - 우리 점수 = 우리 GOAL + 상대 OWN_GOAL / 상대 점수 = 상대 GOAL + 우리 OWN_GOAL.
  - 재계산은 중계 콘솔 흐름에서만(기존 수기 입력 경기 점수는 건드리지 않음).
- 데이터 계약(`live.json`)을 그대로 두면 추후 WebSocket(Channels)로 무리 없이 교체 가능.

## 검증

- `manage.py check` 무이슈.
- Django 테스트 클라이언트로 전체 흐름 검증(트랜잭션 롤백, 실데이터 무변경):
  LIVE 시작 → 우리 득점(+도움)·상대 득점·경고 입력 → 점수 재집계(3:1) →
  `live.json` 200·타임라인 직렬화 확인 → 득점 삭제 시 재집계(2:1) →
  경기 종료 → 비-staff 콘솔 접근 302 리다이렉트.
