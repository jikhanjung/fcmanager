# 20260616_P02 — Phase 4 실시간 중계 MVP (폴링 기반)

## 목표

경기 진행 중, 운영진이 모바일에서 빠르게 이벤트(득점·카드·교체)를 입력하면
공개 경기 상세 페이지의 스코어·타임라인이 **자동으로 갱신**되도록 한다.

## 설계 결정

- **전송 방식: 폴링(Polling)**. WebSocket/Channels/Redis 미도입.
  - 현재 운영: 단일 컨테이너 + Gunicorn(WSGI) + SQLite + nginx 서브패스(`/FcSky/`).
  - 동시 시청자가 소수인 동네 축구팀 규모 → 새 인프라(ASGI 전환·Redis·nginx WS 프록시)
    부담을 지지 않고 기존 스택 그대로 구현 가능.
  - 공개 페이지가 LIVE일 때만 `live.json`을 ~7초 주기로 fetch. 종료되면 폴링 중지.
  - 확장이 필요해지면 동일한 데이터 계약(`live.json` payload)을 WebSocket으로 교체 가능.

- **점수의 단일 출처(Source of Truth)는 `Match.our_score`/`opponent_score`**.
  - 중계 콘솔에서 득점 이벤트를 추가/삭제할 때마다 이벤트 집계로 점수를 **재계산해 저장**.
  - 우리 점수 = (우리 GOAL) + (상대 OWN_GOAL), 상대 점수 = (상대 GOAL) + (우리 OWN_GOAL).
  - 순위표·결과·득점 순위 등 기존 화면이 모두 `Match.our_score`를 쓰므로 일관성 유지.
  - 재계산은 **중계 콘솔 흐름에서만** 수행(기존 수기 입력 경기 점수는 건드리지 않음).

## 범위 (MVP 한 사이클)

1. **운영진 중계 콘솔** `/matches/<pk>/live/` (staff 전용, 모바일 친화)
   - LIVE 시작 / 경기 종료 토글
   - 빠른 입력: ⚽우리 득점(+도움) · ⚽상대 득점 · 🟨경고 · 🟥퇴장 · ⏱교체
   - 분(minute)은 킥오프 기준 경과시간으로 자동 채움(수정 가능)
   - 입력된 이벤트 목록 + 개별 삭제
   - 액션은 일반 POST(액션별 폼) → 새로고침. JS 빌드 도입 없음.
2. **공개 폴링 엔드포인트** `/matches/<pk>/live.json` (읽기 전용)
   - `{status, our_score, opponent_score, is_live, timeline[], ...}`
   - 타임라인은 기존 `_build_timeline`(득점-도움 페어링)을 재사용해 직렬화.
3. **공개 경기 상세** LIVE 인디케이터(🔴 LIVE) + 폴링 JS로 스코어·타임라인 자동 갱신.
4. **일정·결과 목록** LIVE 경기에 🔴 배지.
