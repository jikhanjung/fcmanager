# 20260616_036 — 중계 콘솔 자동 경기 시계(mm:ss)

## 한 일

라이브 중계 콘솔의 분 입력칸(수동)을 **자동으로 흐르는 mm:ss 경기 시계**로 교체했다.
운영자가 분을 직접 칠 필요 없이, LIVE 동안 시계가 1초마다 증가하고 이벤트 기록 시
현재 분이 자동으로 들어간다.

### v0.4.1 — mm:ss 자동 타이머
- `templates/matches/match_live.html`
  - 상단 바의 `<input type=number>` → **mm:ss 시계 버튼**(`#clock`)으로 교체.
  - LIVE 동안 250ms 간격으로 갱신, 매 액션 응답의 `elapsed_seconds`로 재동기화
    → 클라이언트 시계 드리프트 누적 없음.
  - **이벤트 분 규칙: `00:00~00:59 → 1'`** (1-based, `floor(초/60)+1`). 킥오프 전(음수)이면 빈 값.
  - 시계를 **탭하면 분:초 수동 보정**(보정 후에도 계속 흐름).
- `apps/matches/views.py`
  - `_elapsed_seconds(match)` 헬퍼 추가, `_live_payload`·콘솔 GET 컨텍스트에 `elapsed_seconds` 제공.
  - 기존 `suggested_minute`(서버 1회 채움) 제거.

### v0.4.2 — '라이브 시작' 시각 기준으로 보정
- 문제: 타이머 기준이 **예정 킥오프**라, 킥오프가 미래/실제 시작과 다르면 경과가
  음수가 되어 `00:00`에 고정됨(테스트 중 발견 — 6/21 경기를 미리 LIVE 시 멈춰 보임).
- 해결: `Match.live_started_at` 필드 추가(마이그레이션 `0006`).
  - `🔴 LIVE 시작` 누른 **실제 시각**을 기록 → 예정 킥오프와 무관하게 그 시점부터 0:00 시작.
  - 이미 값이 있으면 보존(종료 후 재개 시 시계 연속성 유지).
  - `_elapsed_seconds`는 `live_started_at` 우선, 없으면 킥오프로 폴백(시작 전 표시용).

## 검증

- `0.4.2` 이미지 빌드 후 컨테이너 재기동 → entrypoint가 `0006_match_live_started_at` **자동 적용** 확인.
- staff 로그인 테스트 클라이언트로 콘솔 렌더 200, `#clock`·1-based 분 계산·`syncClock` 연결 확인.
- 미래 킥오프(6/21) 경기 `start` → `live_started_at`=현재시각, `elapsed_seconds=0` 확인
  (이제 0:00부터 증가). 검증 후 6/21 경기 4건은 예정 상태로 복원.

## 배포 메모

- 마이그레이션은 컨테이너 시작 시 entrypoint(`migrate --noinput`)가 자동 적용 — 수동 migrate 불필요.
- 이전에 잘못 LIVE였던 경기는 `live_started_at`이 비어 킥오프 폴백으로 표시됨.
  정상 시계가 필요하면 **종료 → 재개**(또는 시작)를 한 번 누르면 그 시각이 기준으로 잡힌다.
