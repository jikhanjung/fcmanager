# 20260617_060 — SaaS Phase D: 브랜딩 분리

> 선행: Phase C. 이로써 Phase 6(멀티테넌트 SaaS 최소 전환) A~D 완료.

## 한 일

하드코딩된 'FC Sky' 브랜딩을 **`current_club` 기반**으로 — 클럽마다 자기 이름/로고 노출.

- **base.html**: navbar 브랜드(클럽 로고+이름 / 플랫폼은 FC Sky 폴백), `<title>` 기본값,
  footer 를 `current_club` 기준으로.
- **teams/home.html**: 클럽 홈 히어로 로고·제목·문구를 `current_club.name`/`current_club.logo` 로.
- **child 타이틀 22개**: `— FC Sky` → `— {{ current_club.name|default:"FC Sky" }}`.
- 플랫폼(테넌트 밖) 랜딩·브랜드 폴백은 "FC Sky" 유지(플랫폼 자체 이름).

## 검증
- `check` 무이슈, **전체 28개 테스트 통과**.
- 브랜딩 스모크: `데모유나이티드` 클럽 홈 → 'FC SKY' 워드마크 없음·클럽명 표시,
  `fcsky` 홈·플랫폼 랜딩 → 'FC Sky' 유지.

## Phase 6 완료 (멀티테넌트 SaaS 최소 전환)
- A 테넌트 모델 + backfill / B 라우팅 + 스코핑(B1~B5) / C 권한 + 온보딩 / **D 브랜딩 분리** — 전부 완료.
- 여러 클럽이 각자 가입·운영·격리된 채 동작하며, 각 `/<slug>/` 에서 자기 브랜딩으로 표시.

## 운영 배포 전환 (남은 운영 작업, 일괄)
- `DJANGO_URL_PREFIX` 제거(슬러그가 접두사 역할 대체).
- nginx: `/FcSky/` → `/fcsky/` 301 리다이렉트 + static/media alias 를 슬러그 밖(`/static/`,`/media/`)으로.
- 이미지 빌드(버전업) → push → `/srv/FcSky/deploy.sh` 로 교체(엔트리포인트 migrate 자동 적용).
