## 배경 / 이유

기존 부트스트랩 비번(`fcsky1234`)이 git 커밋된 compose 파일에 평문으로 들어 있어
공개 repo 에서 그대로 노출됐다 → 운영 계정 비번을 교체.

## compose 평문 비번 제거 (완료)

`deploy/host/docker-compose.yml`·`deploy/docker-compose.yml` 의
`DJANGO_SUPERUSER_PASSWORD: "fcsky1234"` 라인을 **삭제**하고, 비번은 `env_file`(.env)에서
주입하도록 변경(username 은 비밀 아님 → compose 유지).

- entrypoint 는 계정이 없을 때만 superuser 를 만들므로, 이 변수는 **신규 DB 부트스트랩
  전용**이다. 기존 운영 `admin`(이미 존재, 비번 교체됨)에는 영향 없음.
- compose 에서 라인을 빼도 `env_file: - .env` 가 그대로 로드하므로, `.env` 에
  `DJANGO_SUPERUSER_PASSWORD=...` 가 있으면 부트스트랩에 쓰인다. 없으면 superuser
  생성 단계만 skip(기존 계정 무관).
- 실제 운영 비번은 문서·git 어디에도 평문으로 적지 않는다.

### 운영자 할 일 (선택)

신규 DB 부트스트랩 시 admin 자동 생성을 원하면 `/srv/FcSky/.env` 에 한 줄 추가:
`DJANGO_SUPERUSER_PASSWORD=<원하는값>` (로컬 개발은 `deploy/.env`). 현 운영은
admin 이 이미 있어 추가하지 않아도 무방.

### 남은 평문 참조

- `CLAUDE.md` 의 "개발용 관리자 계정: admin / fcsky1234" 는 **로컬 개발 기본값** 안내.
  운영과 무관하나, 신경 쓰이면 별도로 정리 가능.
