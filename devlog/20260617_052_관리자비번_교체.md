## 배경 / 이유

기존 부트스트랩 비번(`fcsky1234`)이 git 커밋된 compose 파일에 평문으로 들어 있어
공개 repo 에서 그대로 노출됐다 → 운영 계정 비번을 교체.

## 남은 정리 (권장)

- `deploy/host/docker-compose.yml`·`deploy/docker-compose.yml` 의
  `DJANGO_SUPERUSER_PASSWORD` 는 **신규 DB 부트스트랩용 기본값**일 뿐이다.
  entrypoint 는 계정이 없을 때만 superuser 를 만들므로, 이 값이 평문으로 남아 있어도
  **이미 존재하는 운영 `admin` 비번을 바꾸지는 않는다**(교체된 실제 비번이 유효).
- 다만 위생상, 이 부트스트랩 비번도 평문 커밋을 피하려면 `.env`(비추적)로 옮기거나
  더미 placeholder 로 바꾸는 게 좋다. 신규 DB 로 처음 띄울 때만 의미가 있다.
- 실제 운영 비번은 문서·git 어디에도 평문으로 적지 않는다.

## 참고

- 비번 값 자체는 본 기록에 남기지 않음.
