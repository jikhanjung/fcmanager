# 20260616_040 — 대회 부문(Division) 도입 (Phase 1)

## 배경 / 설계

대회 모델 재설계 논의. 확정된 구조:

```
Competition  + year(연도)                    ← 대회(개최 인스턴스)
   └ Division (부문, 선택적)                  ← 20-30대 / 40대 / 50대 / 오픈  ★신규
        (부문 없는 대회도 있음 — 리그 등)
명단 TeamMembership  → competition(+division) ← 명단은 대회 단위로 꾸려짐
경기 Match           → competition + division(선택)
출전 CompetitionEntry → competition + division(선택)
```

- **부문 = 연령 구분**. 구청장기는 20-30대/40대/50대 부문. `K7`은 부문이 아니라 팀/리그 개념
  (스카이 K7 팀이 구청장기에선 "20-30대 부문"으로 출전, K7리그·시민리그는 별도 대회).
- **나이 자격**: 부문 최소 나이(40대=40, 50대=50) 이상이면 출전. 나이 많은 선수가 낮은 부문
  가능(50대→40대부문), 역은 불가. 20-30대·오픈은 하한 없음. 생년 미상이면 차단 안 함.
- **팀 공개 페이지 명단**: "가장 최근 대회(year·id 내림차순)" 기준.

**단계적 진행** 결정: Phase 1은 구조만 추가하고 **Season 모델은 유지**(아카이브·필터 그대로
동작). Phase 2에서 시즌 필터/아카이브를 연도+대회 기준으로 바꾸고 Season을 제거 예정.

## 한 일 (Phase 1)

- 모델
  - `Competition.year` 추가(default 2026).
  - `Division` 신규: competition + age_group(20-30대/40대/50대/오픈) + name. `min_age`·`label`·
    `is_eligible(player)` 헬퍼. unique(competition, age_group).
  - `Match.division`(SET_NULL, nullable) 추가.
  - `CompetitionEntry.division` 추가, unique → (team, competition, division), season nullable화.
  - `TeamMembership`: `competition`+`division` 추가, unique → (player, team, competition). season 유지(Phase 1).
- 마이그레이션
  - competitions 0003(스키마), matches 0007, teams 0004(스키마),
    competitions 0004(**데이터 백필**): 다중 연령대 출전 대회에만 부문 생성, 경기·출전·명단에
    division/competition 귀속. 명단은 '부문 있는 최근 대회'로 귀속.
- 코드
  - teams/views: `_latest_competition`/`_division_for` 헬퍼, team_detail 최신 대회 명단,
    player_add가 명단을 최신 대회+부문에 귀속.
  - teams/forms `MembershipAddForm`: season→competition 기준 제외/검증 + 나이 자격 경고.
  - admin: Division 등록(대회 인라인+독립), Match/Entry/Membership에 competition·division.
  - dedupe_members: 멤버십 중복 판정 season→competition.
  - seed / load_seocho_knockout: 부문 생성·귀속 반영(마이그레이션과 동일 규칙).
  - 템플릿: team_detail에 "○○ 명단" 대회 표시, match_detail에 부문 표시.
