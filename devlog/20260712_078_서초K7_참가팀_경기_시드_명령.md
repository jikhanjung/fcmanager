# 서초 K7 참가팀·경기 시드 명령 (seed_seocho_k7)

2026-07-12

## 목표

`docs/seocho-k7-schedule.md` 의 서초구 K7 참가팀 6팀·HUMBLE 대진표 5경기에
7/12(일) 11:30 스카이 vs 시누쓰 경기를 더해 DB 에 일괄 입력하는 스크립트.

## 내용

`apps/matches/management/commands/seed_seocho_k7.py` — 멱등 관리 커맨드.

- 대상(옵션으로 변경 가능): 클럽 `fcsky` / 팀 `sky-k7` / 대회 `k7-seocho-2026`.
- **참가팀**: 우리 팀(스카이 K7) + 외부팀 5팀(HUMBLEFC·FC오키나와·시누쓰·리얼FC·ACI)을
  `CompetitionEntry` 로 등록. 외부팀은 이름으로 매칭(기존 Opponent 재사용, 없으면 생성),
  약칭(short_name)이 비어 있으면 docs 표의 약칭을 채움. 정식 명칭은 entry 비고에 기록.
- **경기 6건**: 6/28 스카이-HUMBLE, 7/12 스카이-시누쓰(11:30)·HUMBLE-ACI(12:30),
  8/1 HUMBLE-시누쓰, 9/5 리얼-HUMBLE, 9/19 오키나와-HUMBLE. KST 로 파싱해 aware 저장.
  - 생성 시 상태: 킥오프가 과거면 종료(FINISHED), 미래면 예정(SCHEDULED).
  - 재실행 시 같은 대진·같은 날짜 경기와 매칭 → 일시·장소 갱신 + **빈 스코어 채움**,
    수동 입력된 결과·상태는 보존.
  - 결과 반영: **6/28 스카이 1:1 HUMBLE 무**, **7/12 스카이 2:0 시누쓰 승**
    (시드 데이터에 스코어 포함). 7/12 스카이전 장소도 인재개발원으로 확인.
- `--dry-run` 지원(트랜잭션 롤백).

## 검증 (dev_data 사본 DB)

dry-run → 실제 실행 → 재실행(모두 "이미 있음", 신규 0) 확인. 킥오프 UTC 저장값
(13:40 KST→04:40 UTC 등) 및 대회 상세 페이지에 참가팀 6팀·경기 6건 렌더 확인.

## 남은 것

- 스코어 미확보 경기(7/12 HUMBLE-ACI) — 대회 상세 '편집'에서 입력 필요.
- 운영 반영: `docker exec <컨테이너> python manage.py seed_seocho_k7`.
  (dev_data 미러는 매일 05시 운영 백업으로 덮어써지므로 운영에 직접 실행할 것.)
