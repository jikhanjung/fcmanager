# 경기 표시 홈/원정 기준 전환 (None vs None 수정)

2026-07-13

## 문제

서초 K7 시드 후 홈 화면 '다가오는 경기/최근 결과'와 경기 상세가 **None vs None** 으로
표시. 원인: 공개 템플릿들이 우리 팀 관점 호환 프로퍼티(`our_team`/`opponent`/
`our_score`)로 렌더링하는데, 참가팀 개편 후 생긴 **상대팀 간 경기**(HUMBLE vs ACI 등)는
우리 팀 entry 가 없어 모두 None.

## 방침 (사용자 결정)

공개 화면의 경기 표시는 `our_team`/`opponent` 관점을 버리고 **홈 vs 원정** 대칭 표기로
통일. 우리 팀 여부는 각 경기에서 home/away 를 확인해(`is_home`) **굵게 강조**하고,
승/무/패 배지는 우리 팀이 참가한 경기에만 표시. `our_*` 프로퍼티는 내부 용도
(전적 집계 `club_record`, 결과 판정 `result`, 중계 콘솔)로만 유지.

## 변경

- **템플릿(홈/원정 표기 + 우리 팀 굵게)**: `teams/home.html`, `teams/team_detail.html`,
  `matches/match_list.html`(테이블 컬럼 홈|스코어|원정, 모바일 카드 2줄),
  `competitions/year_detail.html`, `teams/player_detail.html`(득점 기록 → `{{ e.match }}`).
- **`matches/match_detail.html`**: 스코어보드를 홈(왼쪽)·원정(오른쪽) 고정으로 전환,
  스코어·승부차기 `home:away`, 제목 `{{ match }}`. 승/무/패·승부차기 승패 배지는
  `is_our_match` 일 때만. staff 버튼: 우리 경기 → 명단/콘솔/결과편집,
  상대팀 간 경기 → 결과 입력(opponent_edit).
- **라이브 JSON**(`_live_payload`): `home_score`/`away_score` 추가(상세 페이지 폴링용).
  `our_score`/`opponent_score` 는 중계 콘솔용으로 유지.
- **뷰 필터**: 홈(`teams.views.home`)·연도 아카이브(`year_detail` recent)의 경기 목록에
  '우리 팀 참가 경기만' 필터 추가(일정&결과와 동일:
  `Q(home_entry__team__isnull=False) | Q(away_entry__team__isnull=False)`).
  상대팀 간 경기는 대회 상세에서만 노출.

## 검증

테스트 50건 통과 + 시드 DB 렌더 검증: 홈(None 없음·우리 경기만·2:0), 상대팀 간 경기
상세(홈/원정 이름), 우리 경기 상세(2:0 + 승리 배지), 일정&결과·연도 아카이브·대회 상세
모두 None 없음. 중계 콘솔 JS 는 유지된 `our_score` 필드 사용 — 무영향.
