# 20260616_042 — 대회 부문 개편 Phase 2: Season 제거 + 연도 기준 전환

## 한 일

[[20260616_040_대회부문_Division_도입_P1]]의 Phase 2. **Season 모델을 완전히 제거**하고
시즌에 얹혀 있던 필터·아카이브를 모두 **연도(Competition.year)+대회** 기준으로 전환했다.

### 모델 — season 필드/모델 제거
- `Match.season`, `OpponentMatch.season`, `TeamMembership.season`,
  `CompetitionEntry.season`, `Award.season` 제거.
- **`Season` 모델 삭제.** Award/Entry ordering을 `competition__year` 기준으로.
- 마이그레이션: teams 0005, matches 0008, competitions 0005(Season 삭제). 순서 자동 해결
  (FK 제거 → 모델 삭제).

### 집계/뷰 — season_id → competition__year
- `services.finished_matches(year)`·`our_events(year)` 연도 인자로 변경.
- matches `schedule`/`scorers`/`stats`: 시즌 필터 → **연도 필터**(`competition__year`),
  드롭다운은 대회 연도 distinct.
- competitions `standings`: 시즌 필터 제거(대회 선택이 연도를 포함). `awards`: 연도 필터 +
  **대회별 그룹**.
- **시즌 아카이브 → 연도 아카이브**: `season_index/detail` → `year_index/detail`
  (`years/`, `years/<year>/`). 연도별 종료 경기·대회·입상 요약, 연도 종합(클럽·입상·득점/도움
  TOP·최근 경기).

### 템플릿
- standings/awards/match_list/scorers/stats: 시즌 select → 연도 select(또는 제거).
- `year_index.html`/`year_detail.html` 신규, `season_index/detail.html` 삭제.
- base.html "시즌 아카이브" → "연도 아카이브". player_detail 소속 이력 시즌 열 → 대회·부문.

### 관리 명령
- seed: Season 생성·season= 제거, 대회에 year 설정.
- import_roster: `--season` → `--competition`(미지정 시 팀 최근 대회), division 자동 설정.
- load_seocho_knockout: Season 생성 제거, 대회 year 설정.
- competitions/tests: Season 제거 반영.
