# 배포·데이터 계약 외부 검토분 정렬 (rollback --db 분리 등 4항목)

- 날짜: 2026-07-14
- 근거: `../devdocs/wiki/deploy-data-contract.md` 2026-07-13 외부 검토 반영분.
  cdGTS 가 먼저 코드로 착지(cdGTS devlog 145, 0.1.61) — 본 작업은 그 **동형 이식**.
  계약 문서가 "fsis2026·fcmanager 는 아직 구계약(rollback = 이미지 전환 + DB 복원 묶음)"으로
  명시하던 갭을 fcmanager 쪽에서 해소.

## 반영 4항목

1. **rollback 코드/DB 분리** (`deploy/host/rollback.sh` 재작성)
   - `--db=keep`(기본) = 이전 이미지 태그로만 전환, 현재 DB 유지(운영 데이터 보존).
   - `--db=restore` = 컨테이너 정지 → 이미지 전환 → pre_deploy 스냅샷 복원(WAL torn-copy 방지).
   - 구계약(복원 묶음)은 배포 후 운영자 입력분까지 지워 rollback 자신이 불변식
     ("파이프라인은 운영 데이터를 나르지도 지우지도 않는다")을 깨는 자기모순이 있었음.
   - **keep 가드**: 직전 배포가 migration 을 적용했으면(스냅샷 `.mig` 사이드카 vs 현재
     `showmigrations` 적용 수 비교) keep 차단 → `--db=restore` 또는 `--force` 승격.
     사이드카 없는 구 스냅샷은 미상으로 두고 진행(대부분 무migration).
2. **pre-deploy 스냅샷 `.mig` 사이드카** (`deploy/host/deploy.sh`)
   - [3/7] down **직전**(정지 후엔 exec 불가) 적용 migration 수 조회 → [4/7] 스냅샷 옆
     `${SNAP}.mig` 로 기록. retention 정리에 `.mig` 포함.
   - cdGTS 와 달리 fcmanager 는 down([3/7])과 스냅샷([4/7])이 분리된 단계라 조회 위치만 다름.
3. **매니페스트 필드** (`deploy/deploy.toml`)
   - `contract_version = 1`, `rollback_db = "keep"` 추가, verbs.rollback 시그니처 갱신.
4. **self-heal 추출 안전망** (`deploy/host/_extract_and_deploy.sh`)
   - 스크립트류(deploy/smoke/rollback/deploy-prod/자기 자신)는 임시 추출 → `bash -n` 통과
     시에만 교체, 기존본은 `<f>.previous` 보존 후 원자 rename. 깨진 스크립트가 이미지에
     실려도 작동하던 부트스트랩 경로는 보존.
   - 비스크립트(docker-compose.yml·backup_db.py)는 종전대로 그대로 추출.
