# 20260620_064 — 경기 목록 행 클릭 + 모바일 카드화 (UI 개선)

> 모바일에서 일정/결과·대시보드가 보기 불편하다는 피드백 반영. 경기 row 전체를
> 클릭 가능하게 하고, 일정/결과 페이지를 모바일에서 카드형으로 재구성.

## 배경

- 폰에서 경기 목록의 "상세" 버튼이 오른쪽 끝에 작게 있어 누르기 어려움.
- 일정/결과 표의 `일시` 컬럼이 `text-nowrap` 이라 화면 폭의 절반을 차지.

## 변경

### 1. 행 전체 클릭 → 경기/공지 상세 이동
- 공통 패턴: 행에 `data-href` + `cursor:pointer` 부여, 작은 스크립트가 행 클릭 시
  `window.location` 이동. 행 안의 `<a>` 를 직접 누르면 기본 동작에 위임
  (`e.target.closest('a')` 체크)로 중복 이동 방지.
- 적용 위치:
  - `templates/matches/match_list.html` — 표 `tr.match-row`. 우측 "상세" 버튼은
    공간을 줄여 `›` 화살표(여전히 링크 → 새 탭/접근성 유지)로 축소.
  - `templates/teams/home.html` — 대시보드의 **공지사항 · 다가오는 경기 · 최근 결과**
    리스트(`.row-link`).
  - `templates/teams/team_detail.html` — 신규 경기 섹션(아래).

### 2. 일정/결과 모바일 카드화 (`templates/matches/match_list.html`)
- 데스크탑(md↑): 기존 표 유지(`d-none d-md-block`).
- 모바일(md↓): 표 대신 경기별 카드(`d-md-none`). 일시는 카드 맨 윗줄 작은 글씨로
  빼고, 팀명은 `text-truncate`, 스코어·승무패 배지는 우측 정렬. 일시가 폭을
  독점하던 문제 해소.

### 3. 팀 상세에 경기 섹션 추가
- `apps/teams/views.py::team_detail` — 해당 팀이 home/away 어느 쪽으로든 참여한
  경기를 조회(`Q(home_entry__team=team) | Q(away_entry__team=team)`)해서
  `upcoming`(예정 5, 취소 제외) / `recent`(종료 5) 컨텍스트 추가.
- `templates/teams/team_detail.html` — 선수단 위에 대시보드와 동일한
  다가오는 경기 / 최근 결과 두 카드. 경기 없으면 섹션 숨김.

### 4. hover 강조 (`static/css/fcsky.css`)
- 클릭 가능한 행에 마우스 오버 시 배경 `#d6e4ff` + 좌측 sky 강조 바(4px).
  `.row-link`(리스트), `tr.match-row`(표 첫 칸), `.match-row.card`(모바일 카드,
  살짝 떠 보이는 그림자 추가)에 적용.

## 검증

  테스트 서버로 폰/좁은 창에서 동작 확인(행 클릭·hover·모바일 카드).
